Cross compiling woe32 binaries on Linux

Obviously you need to install MinGW cross compiler, and (optionally) wine, if you want to test your builds (which is always a good idea). Choosing the right version of cross compiler is very important. You should install MinGW GCC version 3.4, GiNaC is known to not work with MinGW GCC 4.x (where x = 1, 2, 3). You might want to try the binary packages for your favourite distribution if they are based on the stable (3.4) version, however using the official release is preferred. Debian GNU/Linux users, BEWARE! Don't use the mingw32 package shipped in lenny (the latest stable release of Debian). That package is based on the experimental version (4.2.2) of MinGW cross compiler and won't work with GiNaC (and most likely it won't work with any C++ library which uses exceptions to inform the user about error conditions). See e.g. this bug report for more details.

Preparations.

First of all, unpack the tarball somewhere, for instance, in your home directory. If you intend to run the binaries (i.e. to test if they work at all) you need to tell wine where to look for DLLs. Wine tries to mimic Windows' behavior and searches DLLs in the directories listed in the PATH environment variable. Please note: the PATH is wine's own environment variable, NOT the *NIX one. Thus, you should add the $HOME/opt/i586-mingw32msvc/ginac/bin directory to wine's PATH. In order to change wine's PATH edit the ~/.wine/system.reg file with your favourite text editor (search for the "PATH" string), or use the following script:

cp -a ~/.wine/system.reg ~/.wine/system.reg.bak # make a copy if something goes wrong
w_path="$(winepath -w $HOME/opt/i586-mingw32msvc/ginac/bin | sed -e 's/\\/\\\\\\\\/g')"
sed -i -e "s/^\"PATH\"=\(.\+\)\"[ \t]*$/\"PATH\"=\1;${w_path}\"/g" ~/.wine/system.reg

Run the following command to check your setup

wine ginsh.exe

The command above should start the GiNaC shell if your setup is correct.

Cross-compiling a simple program

Run the following commands in order to compile a simple program simple.cpp:

env GINAC_PREFIX="$HOME/opt/i586-mingw32msvc/ginac" \
PKG_CONFIG_LIBDIR="${GINAC_PREFIX}/lib/pkgconfig" \ # (1)
PKG_CONFIG_PATH="" \ # (2)
i586-mingw32msvc-g++ -o simple.exe simple.cpp \ $(pkg-config --define-variable=prefix=${GINAC_PREFIX} --cflags --libs ginac)

  1. Overrides the default path where pkg-config searches for *.pc files. This is necessary to prevent pkg-config from detecting native libraries.
  2. Tell pkg-config to NOT search for *.pc files in other directories. This is necessary to prevent pkg-config from detecting native libraries.
If you've configured wine as described in the previous paragraph you can run the binary:

wine simple.exe

Cross-compiling a package using GiNaC

PKG_CONFIG_PATH="" \
PKG_CONFIG_LIBDIR="$HOME/opt/i586-mingw32msvc/ginac/lib/pkgconfig" \
CC=i586-mingw32msvc-gcc \
CXX=i586-mingw32msvc-g++ \
LD=i586-mingw32msvc-ld \
AS=i586-mingw32msvc-as \
LDFLAGS="-Wl,--enable-auto-import" \
./configure --host=i586-mingw32msvc \--build=--prefix=$HOME/opt/i586-mingw32msvc/foo
